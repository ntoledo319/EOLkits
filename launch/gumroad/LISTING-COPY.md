## Stop tracking AWS EOL deadlines in a spreadsheet.

AWS just synchronized a wave of Lambda runtime blocks into Q1 2027 (Feb 1 create-block, Mar 3 update-block) for
Node.js 18/20 and Python 3.8/3.9/3.10 — and Amazon Linux 2's standard support already ended June 30, 2026. If you
run production workloads on any of these, you have real, current work to do.

This toolkit is the three command-line tools I use to do that migration, packaged with a single playbook that walks
through all three end to end.

**What's inside:**

- **`al2023-gate`** — scans your account for AL2 resources (EC2, launch templates, EKS, ECS, Elastic Beanstalk),
  generates the Packer HCL to rebuild on AL2023, diffs your cloud-init scripts for breakage, rewrites Ansible
  playbooks (yum→dnf), and emits a resource-specific runbook.
- **`python-pivot`** — finds every Python Lambda function on a deprecated runtime, rewrites the known 3.8→3.12
  breakages, audits 30+ common native-wheel dependencies for cp312 compatibility, patches your IaC (SAM/CDK/
  Terraform/Serverless), and runs a staged canary deploy that **refuses to run without a CloudWatch alarm ARN**.
- **`lambda-lifeline`** — same shape for Node.js: finds functions on deprecated runtimes, rewrites `aws-sdk` v2 →
  v3 imports, audits native-binary dependencies for the Node 22 ABI, patches IaC, and gates deploys on an alarm.
- **`MIGRATION-PLAYBOOK.md`** — a single document covering the exact deadlines (sourced straight from AWS's own
  runtime deprecation table, not a blog post that might be citing a superseded date), what breaks in each
  migration, the command sequence for each kit, and a general order-of-operations that applies across all three.

**All three tools are MIT-licensed** — the source is public on GitHub, free forever. What you're buying here is the
packaging: everything in one place, with a playbook that ties the three tools together into one coherent plan,
instead of you piecing it together from three separate READMEs.

**Requirements:** Python 3.9+ (for al2023-gate, python-pivot) and Node.js 18+ (for lambda-lifeline). Live AWS scans
need standard boto3/AWS SDK credentials with read access; all three tools also run in offline fixture mode if you
just want to see the output shape first.

**Full disclosure:** this toolkit and playbook were built with AI assistance (Claude) under human direction — see
`ATTRIBUTIONS.md` inside the download for the complete license and provenance disclosure. No fabricated claims, no
padded feature list — every command listed above is one you can run today.

Need the actual scan run against your account, delivered as a report? See eolkits.com/audit ($299). Need a real
PR opened on your repo with the fixes applied? See eolkits.com/pack ($1,499). Neither is required — this toolkit
is fully self-serve.
